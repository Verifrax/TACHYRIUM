# TACHYRIUM

## Executive statement

TACHYRIUM is the cognition plane of the Verifrax sovereign stack.

It exists to transform ambiguity into bounded, lawful, machine-legible candidates across the Verifrax world-model without becoming law, accepted truth, authority-of-record, governed execution, deterministic verification truth, or active-truth override.

The upper sovereign pattern is:

```text
SYNTAGMARIUM = law
ORBISTIUM    = state
CONSONORIUM  = reconciliation
TACHYRIUM    = cognition
```

## System axiom

```text
TACHYRIUM may think for the stack, but it may never become the stack's law, accepted truth, authority, execution legitimacy, or verification verdict.
```

## Core mission

TACHYRIUM exists to:

* crystallize intent
* compress ambiguity
* understand topology
* diagnose contradiction
* route work lawfully
* generate bounded projections
* amplify operators
* produce structured action candidates

## Non-goals

TACHYRIUM does not exist to:

* define constitutional law
* store accepted world-state
* reconcile reality as sovereign runtime
* issue authority
* execute governed actions
* emit authoritative receipts
* determine verification truth
* publish proof
* replace verifier logic
* own generic docs, status, or intake functions
* become a primitive

## Constitutional boundaries

### TACHYRIUM may:

* read SYNTAGMARIUM law
* read ORBISTIUM state
* read CONSONORIUM contradiction and repair outputs
* read repo, host, package, artifact, and evidence surfaces
* generate bounded drafts
* generate routing proposals
* generate operator summaries
* generate machine-legible candidate structures
* explain ambiguity under existing law

### TACHYRIUM may not:

* override any active-truth surface
* create constitutional exceptions
* mutate accepted epoch truth
* issue authority objects
* act as governed execution runtime
* act as public verifier of record
* act as proof host
* absorb another repo primary role
* convert AI inference into sovereign truth

## Primary capability planes

### 1. Intent plane

Turns raw operator requests into structured bounded candidate forms.

### 2. Topology plane

Understands repo roles, host ownership, package ownership, artifact relationships, and sovereign dependency order.

### 3. Contradiction plane

Detects likely collisions, wording inversions, split truths, and role drift.

### 4. Routing plane

Routes work, people, artifacts, and reasoning targets to the correct lawful surface.

### 5. Projection plane

Produces bounded drafts for README, docs, host copy, indexes, and summaries.

### 6. Amplification plane

Compresses large system complexity into operator-usable structures.
