# TACHYRIUM Runtime Contract

## Purpose

This document defines the runtime boundary for TACHYRIUM.

TACHYRIUM is not a sovereign runtime in the Verifrax sense.
It is a cognition runtime that produces bounded subordinate outputs for human operators and adjacent system surfaces.

## Runtime axiom

```text
ingest -> structure -> analyze -> propose -> stop
```

TACHYRIUM must stop before any boundary that belongs to law, accepted state, authority-of-record, governed execution, enforcement, or final verification truth.

## Runtime role

TACHYRIUM runtime exists to:

* ingest operator intent
* ingest bounded stack context
* normalize inputs into machine-legible structures
* analyze ambiguity, topology, and contradiction
* generate bounded candidate outputs
* route outputs to the correct adjacent surface
* preserve non-overlap with sovereign repos

## Runtime non-role

TACHYRIUM runtime does not:

* legislate
* record accepted truth
* issue authority
* execute governed actions as runtime of record
* emit authoritative receipts
* block merges as the final enforcement gate
* publish proof as source truth
* determine final verification verdicts

## Runtime modes

### 1. intent

Convert raw requests into structured bounded candidate forms.

### 2. topology

Read repo, host, package, artifact, and surface relations.

### 3. contradiction

Detect wording drift, role collision, and split-truth patterns.

### 4. routing

Map work or outputs to the correct lawful surface.

### 5. projection

Generate bounded draft outputs for docs, README, host copy, and summaries.

### 6. operator-brief

Compress complex stack state into reviewable briefings.

## Input contract

Allowed inputs:

* operator prompts
* lawful repo metadata
* accepted state references
* contradiction inventories
* documentation and README surfaces
* host-body text
* package metadata
* evidence trees
* explicitly provided local files

Forbidden inputs if treated as authority:

* AI-generated summaries presented as accepted truth
* undeclared hidden state
* mutable inference treated as law
* repo-local convenience text treated as constitutional override

## Output contract

Allowed outputs:

* structured JSON-like candidate objects
* markdown draft candidates
* contradiction reports
* routing suggestions
* topology summaries
* operator briefings
* bounded analysis notes

Forbidden outputs:

* authoritative truth declarations
* authority objects
* sovereign execution receipts
* merge-gate decisions of record
* final verifier verdicts
* primitive semantics masquerading as canonical truth

## Reviewability rule

Every runtime output must be reviewable as subordinate.

If an output cannot be safely reviewed as subordinate, it is outside the lawful runtime boundary of TACHYRIUM.

## Safety boundary

```text
TACHYRIUM may accelerate interpretation.
TACHYRIUM may not replace sovereign decision surfaces.
```

## Integration rule

Any future implementation inside this repo must preserve:

* one repo, one primary role
* cognition dominance in public messaging
* subordinate output status
* explicit non-overlap with the live Verifrax stack
