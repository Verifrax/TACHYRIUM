# TACHYRIUM Planes

## Purpose

This document defines the internal capability planes of TACHYRIUM.

These planes are capability separations, not sovereign authority claims.

## Plane axiom

```text
capability separation without role expansion
```

## Plane set

### 1. Intent plane

Turns raw operator requests into structured bounded candidate forms.

### 2. Topology plane

Understands repo roles, host ownership, package ownership, artifact relations, and stack dependency order.

### 3. Contradiction plane

Detects wording drift, role collision, split-truth patterns, and adjacency ambiguity.

### 4. Routing plane

Routes work, outputs, and questions to the correct lawful surface.

### 5. Projection plane

Produces bounded draft outputs for README, docs, host copy, indexes, and operator-facing summaries.

### 6. Amplification plane

Compresses large system complexity into reviewable operator briefings.

## Non-overlap rule

These planes do not create new sovereign roles.
They exist only to keep cognition work internally separated and legible.
