# TACHYRIUM Output Contract

## Purpose

This document defines the lawful output boundary for TACHYRIUM.

It specifies which outputs TACHYRIUM may produce, how they must be typed, and which outputs are forbidden because they would collide with sovereign or adjacent Verifrax surfaces.

## Output axiom

```text
every output must be bounded
every output must be subordinate
every output must preserve reviewability
```

## Allowed output classes

TACHYRIUM may emit only bounded outputs such as:

* intent objects
* structured candidate plans
* topology summaries
* contradiction summaries
* routing suggestions
* README draft candidates
* docs draft candidates
* host-copy draft candidates
* operator briefing notes
* analysis artifacts clearly marked subordinate

## Output typing rule

Every output must be recognizable as one of:

* candidate
* draft
* summary
* suggestion
* briefing
* analysis

No output may present itself as sovereign truth.

## Forbidden output classes

TACHYRIUM must never emit outputs that impersonate:

* constitutional law
* accepted epoch truth
* authority-of-record
* governed execution legitimacy
* authoritative receipts
* final merge enforcement
* final verification verdicts
* primitive semantics as canonical fact

## Reviewability rule

Every output must be reviewable by:

* a human operator, or
* the correct adjacent sovereign system

If an output cannot be safely reviewed as subordinate, it is outside the lawful TACHYRIUM output boundary.

## Adjacency rule

Outputs may support:

* SYNTAGMARIUM-facing interpretation
* ORBISTIUM-facing summarization
* CONSONORIUM-facing analysis
* VERIFRAX-facing draft and evidence explanation
* VERIFRAX-SURFACE-facing projection drafts
* governed repo maintenance and clarification work

Outputs may not replace any of those repos as source authority.

## Stability rule

Implementation must conform downward to this contract.

If a future feature requires outputs stronger than this file allows, the feature is outside the current lawful scope of TACHYRIUM.
