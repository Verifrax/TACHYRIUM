# TACHYRIUM Interface Contract

## Purpose

This document defines the first public internal interface boundary for TACHYRIUM.

It specifies what may enter TACHYRIUM, what may leave it, and what must never cross its boundary.

## Interface axiom

```text
input must be explicit
output must be subordinate
boundary crossing must be reviewable
```

## Accepted input classes

TACHYRIUM may accept only bounded inputs such as:

* operator prompts
* explicit repo metadata
* explicit host metadata
* explicit package metadata
* explicit artifact metadata
* explicit evidence files
* accepted state references
* contradiction inventories
* repair-plan references
* local files intentionally provided for analysis

## Rejected input classes

TACHYRIUM must reject or quarantine inputs that attempt to function as:

* constitutional law
* accepted epoch truth
* authority-of-record
* governed execution permission
* final verification verdict
* hidden undeclared state
* ambiguous mixed-trust bundles presented as canonical

## Output classes

TACHYRIUM may emit only bounded outputs such as:

* structured candidate objects
* routing suggestions
* contradiction summaries
* topology summaries
* README draft candidates
* docs draft candidates
* host-copy draft candidates
* operator briefing notes

## Forbidden output classes

TACHYRIUM must never emit outputs that impersonate:

* law
* accepted truth
* authority
* execution legitimacy
* authoritative receipts
* final merge enforcement
* final verification truth

## Review contract

Every output must be reviewable by a human or by the correct sovereign adjacent system.

If an output cannot be reviewed as subordinate, it is outside the lawful TACHYRIUM interface boundary.

## Adjacency contract

TACHYRIUM may interface with:

* SYNTAGMARIUM as law reader
* ORBISTIUM as accepted state reader
* CONSONORIUM as reconciliation-context reader
* VERIFRAX as authored-source and evidence-root reader
* VERIFRAX-SURFACE as projection-adjacent reader
* governed repos as bounded analysis targets

TACHYRIUM may not interface as replacement for any of them.

## Stability rule

This interface contract must remain narrower than the repository mission.
Implementation must conform downward to this file, not expand beyond it by convenience.
